# Copilot 專案規則

## 專案目的
- 主要目的是用來"學習"Vue3 + TypeScript + Vite 的專案架構與開發流程，工程實務取向（而不是刷語法）
- 熟悉 TypeScript 的型別系統與應用，TypeScript 教 常用的為主，高階但常用的也能教
- 清楚知道 JAVA 與 TypeScript 的差異
- 建立一套前端專案的最佳實踐範例，必須嚴格遵守本文件的規則與原則
- 讀者只要讀取此專案，即可對 Vue3 + TypeScript + Vite 有了解，了解程度需達到可以獨立開發中小型專案，這部分最為重要。


## 目標讀者已達成的技能以及了解程度，沒標示的當作不清楚
- 熟悉 HTML、CSS、JavaScript 基礎知識
- 熟悉 Vue3 基礎知識與組件開發
- 了解前端開發流程與工具 (如 Git、npm)
- 了解 JAVA 基礎知識與型別系統，以及以下abstract/extends/interface/override/泛型/型別差異/型別轉換/部分array與部分list差異
- 了解 Vite 的基本概念與使用方式
- 能獨立開發簡單的前端應用程式

## 技術棧
- 提供技術棧選擇 (對於選擇的使用理由需有說明，用註解說明即可)
- 使用 TypeScript + css + Tailwind CSS + HTML
- 樣式原則上使用 Tailwind CSS，必要時可使用 SCSS
- 圖片資源放在assets/images
- css放在assets/css
- js放在assets/js
- 圖標使用font-awesome (4.7.0)
- 為維持穩定，請勿隨意更新套件版本
- 路徑採用相對路徑，以 @ 表示根目錄
- 專案的部分，若有重複性很高的部分，請利用 map 解決
- 降低使用for及forEach，多使用map, filter, reduce等函式式
- 盡量避免使用 var 宣告變數，改用 let 或 const
- 盡量避免使用全域變數，改用區域變數
- 盡量避免使用全域函式，改用區域函式
- 設計.gitIgnore，排除全部專案本身不需要的檔案，不包含 copilot-instructions.md
- 非必要不使用第三方套件，盡量使用原生語法實現功能
- 使用 Composition API，避免使用 Options API

## 命名規則
- 變數名稱：一律採用小駝峰（camelCase），例如：`userName`, `isLoading`
- 常數名稱：全大寫並使用底線（UPPER_CASE_SNAKE），例如：`API_BASE_URL`, `MAX_RETRY_COUNT`
- 命名採用直譯方式，名稱需與用途清楚對應，不使用縮寫或模糊詞彙，例如：
  - ✅ `userEmail`  
  - ❌ `usrEm`

## 組件與檔案命名
- 元件檔案：使用 PascalCase，例如：`UserCard.js`, `LoginForm.js`
- 組件名稱需語意化，避免無意義縮寫，例如：
  - ✅ `ProductList.js`  
  - ❌ `Plist.js`
- 模組化檔案：使用小駝峰（camelCase），例如：`userService.js`, `authUtils.js`

## API 呼叫
- 統一使用 axios，不使用 fetch
- API的使用需有防衛式程式碼
- API 基底網址統一存放在常數，例如：`API_BASE_URL`
- 所有 API 方法需放在 `/src/api/` 目錄下，檔案命名與資源對應，例如：`userApi.ts`

## 其他規範
- 禁止使用 `any`，必須明確定義型別
- 保持程式碼簡潔，減少重複邏輯
- 程式碼的結束，需要添加;
- 避免出現eslint的錯誤
- 程式碼請以更動最少的方式完成需求

- 需考慮效能優化，避免不必要的重複計算與渲染
- 本專案主要是學習用，UI簡單清晰，避免過度設計與複雜化
- 本專案主要是學習用，不需要考慮SEO優化，重點在於程式碼品質與架構設計 (SEO優化的部分，已註解提醒並說明)
- 本專案主要是學習用，要將常見的typescript 與 Vue3的應用場景都涵蓋到，並且在程式碼中註解說明，讓讀者可以從程式碼中學習到這些應用場景 (常見的typescript 與 Vue3的應用場景，已註解提醒並說明)
- 需考慮安全性，避免常見的安全漏洞 (對於常見的安全漏洞，已註解提醒並說明)
- 需考慮跨瀏覽器相容性，確保在主流瀏覽器上正常運作
- 需考量單元測試，確保主要功能有對應的測試案例
- 畫面可簡單呈現即可，重點在於程式碼品質，以及需要達到教學目的

- 確保程式碼品質與可維護性 (包括可讀性、擴展性。擴展性最為重要)
- 註解採一律使用中文
- 由於專案主要是學習用，註解需詳細說明程式碼的邏輯與目的，讓讀者可以從註解中學習到程式碼的設計思路與實現細節
